# 프로세스 vs 스레드

## 1. 프로세스와 스레드

> **Q:** CPU는 한 번에 하나의 일만 할 수 있다는데, 어떻게 여러 프로그램이 동시에 돌아가는가?

> **A:** 운영체제가 **시분할(Time Sharing)** 기법을 사용하기 때문이다. 아주 짧은 시간 동안 프로세스들을 번갈아 실행하며, 이때 현재 상태를 **PCB(Process Control Block)** 에 저장하고 복구하는 **문맥 교환(Context Switch)** 이 매우 빠르게 일어나 우리 눈에는 동시에 실행되는 것처럼 보인다. (※ 참고: 현대의 멀티코어 CPU는 실제로 여러 프로세스를 물리적으로 동시에(Parallelism) 처리하기도 한다.)

> **Q:** 프로세스와 스레드의 가장 큰 차이점은 무엇인가?

> **A:** 핵심은 **자원 공유의 범위**이다.
> - **프로세스**: 운영체제로부터 자원을 할당받는 독립된 실행 단위이다. 독립된 메모리 영역을 가져 안전하지만, 프로세스 간 통신(IPC) 비용이 비싸다.
> - **스레드**: 프로세스 내에서 실행되는 흐름의 단위이다. **Code, Data, Heap** 영역을 공유하며, 각 스레드는 독립적인 실행을 위해 자신만의 **Stack, Register, PC(Program Counter)** 를 가진다. 자원을 공유하므로 통신이 빠르지만, 공유 자원에 대한 동기화 처리가 필수적이다.
