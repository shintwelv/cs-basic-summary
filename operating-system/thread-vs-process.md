# 프로세스 vs 스레드

## 1. 프로세스와 스레드

> **Q:** CPU는 한 번에 하나의 일만 할 수 있다는데, 어떻게 여러 프로그램이 동시에 돌아가는가?

> **A:** 운영체제가 **시분할(Time Sharing)** 기법을 사용하기 때문이다. 아주 짧은 시간 동안 프로세스들을 번갈아 실행하며, 이때 현재 상태를 **PCB(Process Control Block)** 에 저장하고 복구하는 **문맥 교환(Context Switch)** 이 매우 빠르게 일어나 우리 눈에는 동시에 실행되는 것처럼 보인다. (※ 참고: 현대의 멀티코어 CPU는 실제로 여러 프로세스를 물리적으로 동시에(Parallelism) 처리하기도 한다.)

> **Q:** 프로세스와 스레드의 가장 큰 차이점은 무엇인가?

> **A:** 핵심은 **자원 공유의 범위**이다.
> - **프로세스**: 운영체제로부터 자원을 할당받는 독립된 실행 단위이다. 독립된 메모리 영역을 가져 안전하지만, 프로세스 간 통신(IPC) 비용이 비싸다.
> - **스레드**: 프로세스 내에서 실행되는 흐름의 단위이다. **Code, Data, Heap** 영역을 공유하며, 각 스레드는 독립적인 실행을 위해 자신만의 **Stack, Register, PC(Program Counter)** 를 가진다. 자원을 공유하므로 통신이 빠르지만, 공유 자원에 대한 동기화 처리가 필수적이다.

## 2. 교착 상태 (Deadlock)

> **Q:** 교착 상태란 무엇이며, 왜 발생하는가?

> **A:** 여러 프로세스가 서로 상대방이 가진 자원을 기다리며 무한히 대기하는 상태이다. 다음 **4가지 조건**이 모두 충족될 때 발생한다.
> 1. **상호 배제 (Mutual Exclusion)**: 한 번에 한 프로세스만 자원 사용 가능
> 2. **점유 및 대기 (Hold and Wait)**: 자원을 가진 채 다른 자원을 기다림
> 3. **비선점 (No Preemption)**: 다른 프로세스의 자원을 강제로 뺏을 수 없음
> 4. **환형 대기 (Circular Wait)**: 대기 프로세스들이 고리 형태로 서로를 기다림

> **Q:** 교착 상태를 해결하는 전략은 무엇이 있는가?

> **A:** 
> - **예방(Prevention)**: 발생 조건 4가지 중 하나를 제거한다. 예를 들어, 자원에 번호를 매겨 순차적으로 요청하게 하여 '환형 대기'를 차단할 수 있다. (단, 자원 효율성이 떨어질 수 있음)
> - **회피(Avoidance)**: 자원 할당 전 **은행원 알고리즘** 등을 통해 '안전 순서(Safe Sequence)'가 있는지 확인하여 안전한 경우에만 할당한다.
> - **탐지 및 복구(Detection & Recovery)**: 교착 상태를 허용하되 주기적으로 체크하여 발견 시 프로세스를 종료하거나 자원을 선점하여 해결한다.

## 3. CPU 스케줄링

> **Q:** 먼저 온 작업을 먼저 처리하는(FCFS) 것이 항상 좋은가?

> **A:** 아니다. 실행 시간이 긴 작업이 먼저 오면 뒤의 짧은 작업들이 무작정 기다리는 **호위 효과(Convoy Effect)** 가 발생하여 전체 시스템 효율이 떨어진다.

> **Q:** 그렇다면 짧은 작업을 먼저 처리(SJF)하면 해결되나?

> **A:** 평균 대기 시간은 최소화할 수 있지만, 실행 시간이 긴 작업은 영원히 CPU를 할당받지 못하는 **기아 상태(Starvation)** 에 빠질 수 있다. 이를 방지하기 위해 오래 기다린 프로세스의 우선순위를 점진적으로 높여주는 **에이징(Aging)** 기법을 사용한다.
