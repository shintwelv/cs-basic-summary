# 동기화 (Mutex & Semaphore)

여러 프로세스나 스레드가 공유 자원에 동시에 접근할 때 발생하는 문제를 해결하고, 데이터의 일관성을 유지하기 위한 교통 정리 과정이다.

## 1. 임계 구역 (Critical Section)

공유 데이터에 접근하는 코드 영역으로, 동기화 문제가 발생하는 지점이다. 이를 해결하기 위해 반드시 다음 3가지 조건을 충족해야 한다.

- **상호 배제 (Mutual Exclusion)**: 한 프로세스가 임계 구역에 있으면 다른 프로세스는 들어올 수 없다.
- **진행 (Progress)**: 임계 구역이 비어 있다면, 들어가고자 하는 프로세스 중 하나를 지체 없이 선택해야 한다.
- **한정된 대기 (Bounded Waiting)**: 어떤 프로세스가 무한히 기다리지 않도록, 다른 프로세스가 임계 구역에 들어가는 횟수에 제한을 둔다.

## 2. 뮤텍스 (Mutex)

"단 하나의 열쇠가 있는 화장실" 비유로 설명된다. 상호 배제를 달성하기 위한 가장 기본적인 도구이다.

- 핵심 특징: 소유권(Ownership)이 존재한다.

- 작동 원리:
    - `lock()`: 열쇠를 획득하여 문을 잠근다. (값이 1 -> 0)
    - `unlock()`: 열쇠를 반납하여 문을 연다. (값이 0 -> 1)
- 제약: 반드시 열쇠를 잠근 프로세스만이 그 잠금을 풀 수 있다.

## 3. 세마포어 (Semaphore)

"전광판이 있는 주차장" 혹은 "신호등" 비유로 설명된다. 뮤텍스보다 더 넓은 개념의 도구이다.

- 핵심 특징: 소유권이 없으며, 정수 값 $S$를 이용해 자원을 관리한다.

- 종류:
    - Binary Semaphore: 값이 0과 1뿐인 세마포어 (뮤텍스와 유사하지만 소유권 없음).
    - Counting Semaphore: 자원이 여러 개일 때 사용 (값이 0, 1, 2, ...).
- 작동 원리:
    - `wait(S)` ($P$ 연산): 자원 사용을 요청하며 값을 1 줄인다. 값이 0이면 대기한다.
    - `signal(S)` ($V$ 연산): 자원 사용을 마치고 신호를 보내며 값을 1 늘린다.

- 주요 용도: 자원 개수 관리 및 실행 순서(Ordering) 제어.

## 4. 뮤텍스 vs 세마포어 비교

| 구분 | 뮤텍스 (Mutex) | 세마포어 (Semaphore) |
|---|---|---|
| 소유권 | 있음 (Lock을 건 자만 해제 가능) | 없음 (누구나 신호 가능) |
| 자원 개수 | 무조건 1개 | 1개 이상 가능 |
| 우선순위 상속 | 지원 가능 (OS가 소유자 추적 가능) | 지원 불가 (누가 신호 보낼지 모름) |
| 주 목적 | 데이터 보호 (상호 배제) | 자원 관리 및 작업 순서 제어 |
|자원 개수|무조건 1개|1개 이상 가능|
|우선순위 상속|지원 가능 (OS가 소유자 추적 가능)|지원 불가 (누가 신호 보낼지 모름)|
|주 목적|데이터 보호 (상호 배제)|자원 관리 및 작업 순서 제어

## 5. 고전적 동기화 문제: 생산자-소비자 (Producer-Consumer)

버퍼(창고)를 사이에 두고 데이터를 주고받는 모델이다. 세마포어 3개를 사용해 해결한다.

1. `empty` (세마포어): 비어 있는 칸의 수 (초기값: N). 생산자가 확인.
2. `full` (세마포어): 채워진 칸의 수 (초기값: 0). 소비자가 확인.
3. `mutex` (뮤텍스): 버퍼 자체에 대한 접근 보호 (초기값: 1).

- 생산자: `wait(empty)` -> `wait(mutex)` -> [물건 넣기] -> `signal(mutex)` -> `signal(full)`
- 소비자: `wait(full)` -> `wait(mutex)` -> [물건 빼기] -> `signal(mutex)` -> `signal(empty)`

## 6. 우선순위 역전 (Priority Inversion) & 상속 (Inheritance)

낮은 우선순위의 프로세스($L$)가 자원을 쥐고 있어, 높은 우선순위의 프로세스($H$)가 중간 순위($M$)에게 밀려 실행되지 못하는 현상이다.

- 발생 시나리오: $L$이 자원 점유 -> $H$ 대기 -> $M$이 $L$의 CPU를 뺏음 -> $H$는 $M$이 끝날 때까지 대기.

- 해결책 (우선순위 상속): $H$가 기다리는 자원을 가진 $L$에게 일시적으로 $H$의 우선순위를 빌려준다.

- 특징: 이 해결책은 소유권이 명확한 뮤텍스에서만 가능하다.
