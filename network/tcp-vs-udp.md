# TCP & UDP 및 전송 계층(Transport Layer) 학습 정리

이 문서는 **[OSI 7 계층](./layer.md)** 중 전송 계층(L4)의 핵심 프로토콜인 TCP와 UDP에 대해 정리합니다.

## 1. TCP vs UDP: 철학의 차이

| 특성 | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
| :--- | :--- | :--- |
| **핵심 키워드** | 신뢰성, 연결 지향, 정확성 | 속도, 비연결형, 단순함 |
| **비유** | 꼼꼼한 등기 우편 | 던져두고 가는 신문 배달 |
| **주요 기능** | 흐름 제어, 혼잡 제어, 재전송 | 최소한의 오류 검출 (Checksum) |
| **사용 사례** | HTTP(웹), 파일 전송, 이메일 | 전화 통화(VoIP), 게임(위치 정보), 스트리밍 |

---

## 2. TCP 연결의 시작: 3-Way Handshake

송신자와 수신자가 양방향 통신이 가능한지 확인하고, 데이터의 순서 번호를 맞추는 과정입니다.

1.  **SYN ($x$):** 클라이언트가 서버에게 "내 목소리 들려? 나는 $x$번부터 보낼게."
2.  **SYN ($y$) + ACK ($x+1$):** 서버가 클라이언트에게 "응 잘 들려! 나는 $y$번부터 보낼게. 너는 이제 $x+1$번 보낼 차례야."
3.  **ACK ($y+1$):** 클라이언트가 서버에게 "확인했어! 너는 이제 $y+1$번 보낼 차례야. 이제 진짜 데이터 보낸다!"

> **학습 포인트:** 3번의 과정을 거치는 이유는 양방향(Full-Duplex) 통신에서 서로의 송신 및 수신 능력을 모두 확인하기 위함입니다.

## 3. TCP 흐름 제어 (Flow Control)

수신자의 처리 속도에 맞춰 송신자의 전송량을 조절하여 버퍼가 넘치는 것(Overflow)을 방지합니다.

-   **슬라이딩 윈도우 (Sliding Window):** 수신자가 헤더의 `Window Size` 필드를 통해 자신의 가용 용량($RWND$)을 알려주면, 송신자는 그 범위 안에서 데이터를 연속해서 보냅니다.
-   **제로 윈도우 프로브 (Zero Window Probe):** 수신자의 여유 공간이 0이 되어 통신이 멈췄을 때, 교착 상태(Deadlock)를 방지하기 위해 송신자가 주기적으로 수신자의 상태를 확인하는 'Health Check' 메커니즘입니다.

## 4. TCP 혼잡 제어 (Congestion Control)

네트워크 도로의 정체 상황을 파악하여 전체 네트워크의 마비를 방지합니다.

-   **느린 시작 (Slow Start):** 처음에는 1개부터 시작하여 성공할 때마다 전송량을 지수적으로($1, 2, 4, 8 \dots$) 늘립니다.
-   **혼잡 회피 (Congestion Avoidance):** 임계점($ssthresh$)에 도달하면 전송량을 선형적으로($+1$) 조심스럽게 늘립니다.
-   **AIMD (합 증가/곱 감소):**
    -   문제가 없을 땐 조금씩 늘리고($+1$),
    -   중복 ACK 3번 발생 시(정체 의심) 전송량을 절반으로 줄입니다 (빠른 회복).
    -   타임아웃 발생 시(마비 판단) 전송량을 다시 1로 줄이고 처음부터 다시 측정합니다.
