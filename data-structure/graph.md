# 그래프 (Graph)

## 그래프의 구성과 특징

### 구성 요소
- 정점 (Vertex)
- 간선 (Edge)

## 그래프의 구현 방식

| 구분 | 인접 행렬 (Adjacency Matrix) | 인접 리스트 (Adjacency List) |
|---|---|---|
|방식|2차원 배열 사용 (`adj[i][j]`)|연결 리스트 혹은 가변 배열 사용`|
|공간 복잡도|$O(V^2)$|$O(V+E)$ (정점 + 간선 수)|
|장점|두 정점의 연결 여부를 $O(1)$ 에 확인 가능|메모리 효율적 (특히 간선이 적은 희소 그래프)|
|단점|간선이 적을 때 메모리 낭비가 심함|두 정점의 연결 확인을 위해 리스트 탐색 필요|

## 그래프 탐색

### BFS (너비 우선 탐색)
- 철학: 시작점에서 가까운 노드부터 차례대로 방문 (파동이 퍼지듯 탐색)
- 자료구조: 큐 (Queue) 사용 (먼저 발견한 것을 먼저 방문)
- 주요 용도: 가중치가 없는 그래프에서의 최단 경로 찾기

### DFS (깊이 우선 탐색)
- 철학: 한 방향으로 갈 수 있을 때까지 끝까지 파고든 후 돌아옴
- 자료구조: 스택 (Stack) 또는 재귀함수 사용
- 주요 용도: 모든 노드를 방문해야 할 때, 경로의 특징을 저장해야 할 때, 사이클 감지

## 가중치 그래프와 최단 경로

- 완화 (Relaxation): 현재 알고 있는 거리보다 더 짧은 경로를 발견하면 정보를 업데이트
- 다익스트라 (Dijkstra)
    - 가중치가 양수인 그래프가 시작점으로부터 다른 모든 노드까지의 최단 경로를 산정
    - 탐욕적 (Greedy) 선택: 방문하지 않은 노드 중 '현재 가장 거리가 짧은 노드'를 우선 선택
    - 효율성: 최소값을 빠르게 찾기 위해 **우선순위 큐 (Priority Queue / Heap)** 를 사용하는 것이 유리
