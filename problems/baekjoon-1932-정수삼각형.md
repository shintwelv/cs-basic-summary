# 백준 1932번 - 정수 삼각형

https://www.acmicpc.net/problem/1932

## 제한
시간제한: 2초
메모리제한: 128MB

## 문제
```
        7
      3   8
    8   1   0
  2   7   4   4
4   5   2   6   5
```
위 그림은 크기가 5인 정수 삼각형의 한 모습이다.

맨 위층 7부터 시작해서 아래에 있는 수 중 하나를 선택하여 아래층으로 내려올 때, 이제까지 선택된 수의 합이 최대가 되는 경로를 구하는 프로그램을 작성하라. 아래층에 있는 수는 현재 층에서 선택된 수의 대각선 왼쪽 또는 대각선 오른쪽에 있는 것 중에서만 선택할 수 있다.

삼각형의 크기는 1 이상 500 이하이다. 삼각형을 이루고 있는 각 수는 모두 정수이며, 범위는 0 이상 9999 이하이다.

## 입력
첫째 줄에 삼각형의 크기 n(1 ≤ n ≤ 500)이 주어지고, 둘째 줄부터 n+1번째 줄까지 정수 삼각형이 주어진다.

## 출력
첫째 줄에 합이 최대가 되는 경로에 있는 수의 합을 출력한다.

## 예제

|입력|출력|
|---|---|
|5<br>7<br>3 8<br>8 1 0<br>2 7 4 4<br>4 5 2 6 5|30|

### 1. 문제 분석 및 접근
이 문제는 **정수 삼각형**의 꼭대기에서 아래층으로 내려오며 숫자를 선택해 나갈 때, 경로의 합이 최대가 되는 경우를 찾는 문제다. 

- **제약 조건**: 아래층으로 내려갈 때는 현재 위치에서 대각선 왼쪽 또는 오른쪽으로만 이동할 수 있다.
- **최적 부분 구조 (Optimal Substructure)**: 각 위치에서의 최대 합은 이전 단계(위층)에서의 최적해를 통해 계산될 수 있다. 
- **접근 방식**: 모든 경로를 탐색하는 것은 $2^N$에 가까운 시간이 소요될 수 있으므로, 중복 계산을 피하기 위해 **[다이나믹 프로그래밍(DP)](../algorithm/dp.md)** 을 사용한다. 큰 문제를 해결하기 위해 작은 문제(위층의 최대 합)의 결과를 저장하고 재사용하는 Bottom-up 방식을 적용한다.


### 2. 핵심 알고리즘
- **다이나믹 프로그래밍 (Dynamic Programming)**
  - 각 노드까지의 최대 합을 저장하는 2차원 배열 `route`를 정의한다.
  - **점화식**: `DP[i][j] = triangle[i][j] + max(DP[i-1][j-1], DP[i-1][j])`
    - `j = 0` (맨 왼쪽): `DP[i-1][j]` 값만 참조 가능하다.
    - `j = i` (맨 오른쪽): `DP[i-1][j-1]` 값만 참조 가능하다.
    - 그 외: 두 개의 부모 중 더 큰 값을 선택하여 현재 노드의 값과 더한다.


### 3. 상세 동작 시나리오 (Dry Run)
예제 삼각형의 일부를 예로 들면:
```
    7
  3   8
8   1   0
```

1. **Layer 0**: `[7]`로 초기화.
2. **Layer 1**:
   - `index 0`: `3 + 7 = 10`
   - `index 1`: `8 + 7 = 15`
   - 상위 레이어 결과: `[10, 15]`
3. **Layer 2**:
   - `index 0`: `8 + 10 = 18` (왼쪽 끝)
   - `index 1`: `1 + max(10, 15) = 16` (부모인 10과 15 중 15 선택)
   - `index 2`: `0 + 15 = 15` (오른쪽 끝)
   - 상위 레이어 결과: `[18, 16, 15]`

최종적으로 마지막 층의 배열 중에서 가장 큰 값을 출력하면 전체 경로 중 최대 합이 된다.


### 4. 코드 구현 (Swift)

```swift
// input
let N: Int = Int(readLine()!)!
let triangle: [[Int]] = Array(repeating: 0, count: N).map { _ in return readLine()!.split(separator: " ").map { Int($0)! } }

// solution
var route: [[Int]] = triangle.map { Array(repeating: Int.max, count: $0.count) }
route[0] = triangle[0]

for i in 1..<route.count {
    for j in 0..<route[i].count {
        var candidate: [Int] = []
        
        if (j-1) >= 0 { candidate.append(route[i-1][j-1]) }
        if j < route[i-1].count { candidate.append(route[i-1][j]) }
        
        route[i][j] = candidate.max()! + triangle[i][j]
    }
}

print(route[N-1].max()!)
```

### 5. 복잡도 분석
- **시간 복잡도**: $O(N^2)$
  - 삼각형의 크기 $N$에 대해 전체 요소의 개수는 $N(N+1)/2$이다. 각 요소를 한 번씩 방문하며 상수 시간의 연산(비교 및 더하기)을 수행하므로 $O(N^2)$이다.
- **공간 복잡도**: $O(N^2)$
  - 입력받은 삼각형과 동일한 크기의 `route` 배열을 사용하므로 $O(N^2)$의 공간이 필요하다. (만약 입력 배열을 직접 수정한다면 $O(1)$로 최적화가 가능하다.)
