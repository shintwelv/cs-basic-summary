# 백준 11399번 - ATM

https://www.acmicpc.net/problem/11399

## 문제
인하은행에는 ATM이 1대밖에 없다. 지금 이 ATM앞에 N명의 사람들이 줄을 서있다. 사람은 1번부터 N번까지 번호가 매겨져 있으며, i번 사람이 돈을 인출하는데 걸리는 시간은 Pi분이다.

사람들이 줄을 서는 순서에 따라서, 돈을 인출하는데 필요한 시간의 합이 달라지게 된다. 예를 들어, 총 5명이 있고, P1 = 3, P2 = 1, P3 = 4, P4 = 3, P5 = 2 인 경우를 생각해보자. [1, 2, 3, 4, 5] 순서로 줄을 선다면, 1번 사람은 3분만에 돈을 뽑을 수 있다. 2번 사람은 1번 사람이 돈을 뽑을 때 까지 기다려야 하기 때문에, 3+1 = 4분이 걸리게 된다. 3번 사람은 1번, 2번 사람이 돈을 뽑을 때까지 기다려야 하기 때문에, 총 3+1+4 = 8분이 필요하게 된다. 4번 사람은 3+1+4+3 = 11분, 5번 사람은 3+1+4+3+2 = 13분이 걸리게 된다. 이 경우에 각 사람이 돈을 인출하는데 필요한 시간의 합은 3+4+8+11+13 = 39분이 된다.

줄을 [2, 5, 1, 4, 3] 순서로 줄을 서면, 2번 사람은 1분만에, 5번 사람은 1+2 = 3분, 1번 사람은 1+2+3 = 6분, 4번 사람은 1+2+3+3 = 9분, 3번 사람은 1+2+3+3+4 = 13분이 걸리게 된다. 각 사람이 돈을 인출하는데 필요한 시간의 합은 1+3+6+9+13 = 32분이다. 이 방법보다 더 필요한 시간의 합을 최소로 만들 수는 없다.

줄을 서 있는 사람의 수 N과 각 사람이 돈을 인출하는데 걸리는 시간 Pi가 주어졌을 때, 각 사람이 돈을 인출하는데 필요한 시간의 합의 최솟값을 구하는 프로그램을 작성하시오.

## 입력
첫째 줄에 사람의 수 N(1 ≤ N ≤ 1,000)이 주어진다. 둘째 줄에는 각 사람이 돈을 인출하는데 걸리는 시간 Pi가 주어진다. (1 ≤ Pi ≤ 1,000)

## 출력
첫째 줄에 각 사람이 돈을 인출하는데 필요한 시간의 합의 최솟값을 출력한다.

## 예제

|입력|출력|
|---|---|
|5<br>3 1 4 3 2|32|

### 1. 문제 분석 및 접근

- ATM 앞에 줄을 선 사람들이 돈을 인출하는 데 필요한 총 대기시간의 합을 최소화하는 문제다.
- 각 사람이 기다리는 시간은 **이전 사람들이 인출하는 데 걸린 시간의 누적합**이다.
- 결과적으로, 대기 시간의 총합을 최소화하려면 **인출 시간이 짧은 사람이 먼저 인출하도록 줄을 서야 한다.**
- 즉, 짧은 시간이 여러 번 누적되도록 하고 긴 시간은 적게 누적되도록 최적의 해를 찾는 **[그리디(Greedy) 알고리즘](../greedy.md)** 문제다.

---

### 2. 핵심 알고리즘

1. **정렬 (Sorting)**: 입력받은 인출 시간 배열을 빠른 순서대로(오름차순) 정렬한다. 가장 짧은 시간이 앞쪽에 오게 한다.
2. **수학적 최적화 탐색**: 일반적으로 배열 요소를 누적합(Prefix Sum)하여 모두 더하는 방식도 가능하지만, 각 인출 시간이 **결과에 몇 번 더해지는지** 곱셈 연산을 통해 한 번에 계산했다.
    - 정렬된 배열에서 `i`번째 사람의 인출 시간은 **본인을 포함해 뒤에 대기하는 모든 사람들의 대기 시간에 포함**된다.
    - 따라서 `i`번째 시간은 총 `(N - i)`번 더해지게 된다.

---

### 3. 상세 동작 시나리오 (Dry Run)

- **입력**: `N = 5`, `pis = [3, 1, 4, 3, 2]`
- **정렬 후**: `sortedPis = [1, 2, 3, 3, 4]`
- **초기화**: `result = 0`
- **반복문 (i = 0 ~ 4)**:
    - **i = 0 (`time` = 1)**: `result = 0 + 1 * (5 - 0) = 5` (본인 포함 총 5명에게 1분씩 추가됨)
    - **i = 1 (`time` = 2)**: `result = 5 + 2 * (5 - 1) = 13` (총 4명에게 2분씩 추가됨)
    - **i = 2 (`time` = 3)**: `result = 13 + 3 * (5 - 2) = 22` (총 3명에게 3분씩 추가됨)
    - **i = 3 (`time` = 3)**: `result = 22 + 3 * (5 - 3) = 28` (총 2명에게 3분씩 추가됨)
    - **i = 4 (`time` = 4)**: `result = 28 + 4 * (5 - 4) = 32` (본인 1명에게만 4분 추가됨)
- **최종 결과**: `32` 반환

---

### 4. 코드 구현 (Swift)

```swift
// input
let N: Int = Int(readLine()!)!
let pis: [Int] = readLine()!.split(separator: " ").map {Int($0)!}

// solution
let sortedPis: [Int] = pis.sorted()

var result: Int = 0

for i in 0..<sortedPis.count {
    let time = sortedPis[i]

    result = result + time * (sortedPis.count - i)
}

print(result)
```

### 5. 복잡도 분석

- **시간 복잡도: $\mathcal{O}(N \log N)$**
  - 입력받은 배열을 정렬하는 데 `sorted()` 메서드는 $\mathcal{O}(N \log N)$의 시간이 소요된다.
  - 이후 루프를 1회 순회하며 연산하므로 $\mathcal{O}(N)$이 걸린다.
  - 따라서 가장 높은 차수를 따라 전체 시간 복잡도는 $\mathcal{O}(N \log N)$이다.
- **공간 복잡도: $\mathcal{O}(N)$**
  - 원본 입력 배열 이외에 `sortedPis`라는 정렬된 새로운 배열을 만드므로 길이 $N$에 해당하는 $\mathcal{O}(N)$의 추가 공간을 사용한다.
