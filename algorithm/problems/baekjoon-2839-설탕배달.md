# 백준 2839번 - 설탕배달

https://www.acmicpc.net/problem/2839

## 문제
상근이는 요즘 설탕공장에서 설탕을 배달하고 있다. 상근이는 지금 사탕가게에 설탕을 정확하게 N킬로그램을 배달해야 한다. 설탕공장에서 만드는 설탕은 봉지에 담겨져 있다. 봉지는 3킬로그램 봉지와 5킬로그램 봉지가 있다.

상근이는 귀찮기 때문에, 최대한 적은 봉지를 들고 가려고 한다. 예를 들어, 18킬로그램 설탕을 배달해야 할 때, 3킬로그램 봉지 6개를 가져가도 되지만, 5킬로그램 3개와 3킬로그램 1개를 배달하면, 더 적은 개수의 봉지를 배달할 수 있다.

상근이가 설탕을 정확하게 N킬로그램 배달해야 할 때, 봉지 몇 개를 가져가면 되는지 그 수를 구하는 프로그램을 작성하시오.

## 제한사항
|시간제한|메모리제한|
|-----|-------|
|1초|128MB|

## 입력
첫째 줄에 N이 주어진다. (3 ≤ N ≤ 5000)

## 출력
상근이가 배달하는 봉지의 최소 개수를 출력한다. 만약, 정확하게 N킬로그램을 만들 수 없다면 -1을 출력한다.

## 예제

|입력|출력|
|---|---|
|18|4|
|4|-1|
|6|2|
|9|3|
|11|3|

### 접근 방식: 동적 계획법 (Dynamic Programming)
이 문제는 더 작은 단위(3kg, 5kg)의 해를 조합하여 큰 단위의 해를 구하는 전형적인 Bottom-Up DP 문제이다. [그리디(Greedy) 알고리즘](../greedy.md)으로도 풀 수 있지만, [DP](../dp.md)를 사용하면 모든 경우의 수를 안정적으로 고려할 수 있다.

### 상태 정의 및 점화식
- 상태 정의: dp[i]를 $i$ kg 설탕을 배달하기 위해 필요한 최소 봉지 개수로 정의한다.
- 점화식: $i$ kg에 도달하는 방법은 $(i-3)$ kg에서 3kg 봉지를 1개 추가하거나, $(i-5)$ kg에서 5kg 봉지를 1개 추가하는 두 가지 경로뿐이다. $$dp[i] = \min(dp[i-3], dp[i-5]) + 1$$ (단, $dp[i-3]$ 또는 $dp[i-5]$가 가능한 경우에만 적용)

### 주요 로직 설명
- 배열 크기 설정: max(6, N + 1)을 사용하여 $N$이 매우 작을 때 발생하는 Index Out of Range 에러를 방지했다.
- 필터링 로직: `candidates.filter({ $0 > -1 })`를 통해 만들 수 없는 상태(-1)를 계산에서 제외하고, 가능한 후보들 중에서만 min()을 구했다.
- 효율성: 루프 한 번($O(N)$)으로 정답을 유도하므로 시간 제한 내에 충분히 통과할 수 있다.

```swift
import Foundation

//input
let N: Int = Int(readLine()!)!

// solution
var dp: [Int] = Array(repeating: -1, count: max(6, N+1))

dp[0] = 0
dp[1] = -1
dp[2] = -1
dp[3] = 1
dp[4] = -1
dp[5] = 1

if N >= 6 {
    for i in 6..<(N+1) {
        let minus3 = dp[i-3]
        let minus5 = dp[i-5]

        let candidates: [Int] = [minus3, minus5].filter({$0 > -1}).map({$0 + 1})
        dp[i] = candidates.min() ?? -1
    }
}
print(dp[N])
```
