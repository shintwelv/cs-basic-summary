# 검색 (Search)

| 알고리즘 | 데이터 상태 | 시간 복잡도 | 핵심 아이디어 |
| :--- | :--- | :--- | :--- |
| 선형 검색 | 아무 상태나 | O(n) | 처음부터 끝까지 하나씩 확인 |
| 이진 검색 | 정렬 필수 | O(logn) | 중간을 찍고 범위를 절반씩 제거 |
| 해시 테이블 | 상관없음 | O(1) | 함수로 주소를 계산해 바로 접근 |

## 1. 선형 검색 (Linear Search)

> **Q:** 선형 검색의 원리와 장단점은 무엇인가?
>
> **A:** 배열의 처음부터 끝까지 순차적으로 탐색하는 방식이다.
> - **장점:** 구현이 매우 간단하고, 데이터가 정렬되어 있지 않아도 사용할 수 있다.
> - **단점:** 최악의 경우 배열의 모든 요소를 탐색해야 하므로 시간 복잡도가 O(n)으로 비효율적이다.

## 2. 이진 검색 (Binary Search)

> **Q:** 이진 검색의 원리와 장단점은 무엇인가?
>
> **A:** 정렬된 배열에서 중간값을 기준으로 탐색 범위를 절반씩 줄여나가는 방식이다.
> - **장점:** 시간 복잡도가 O(logn)으로 매우 효율적이다.
> - **단점:** 데이터가 정렬되어 있어야 사용할 수 있다.

## 3. 해시 테이블 (Hash Table)

> **Q:** 해시 테이블의 원리와 장단점은 무엇인가?
>
> **A:** 해시 함수를 사용하여 키를 배열의 인덱스로 변환하여 데이터를 저장하고 탐색하는 방식이다.
> - **장점:** 평균적으로 시간 복잡도가 O(1)으로 매우 빠르다.
> - **단점:** 최악의 경우 시간 복잡도가 O(n)이 될 수 있다.

### 3-1. 해시 충돌 (Hash Collision)

> **Q:** 해시 충돌이란 무엇인가?
>
> **A:** 서로 다른 키가 동일한 해시 값을 가지는 현상이다.

> **Q:** 해시 충돌을 해결하는 방법은 무엇인가?
>
> **A1:** 연결 리스트를 사용하여 충돌된 데이터를 저장한다. (체이닝 / Chaining)
> - **장점:** 데이터 삽입에 제한이 없다
> <br> 해시 함수가 데이터를 완벽하게 분산시키지 못하더라고 성능이 급격히 무너지지 않고 비교적 안정적이다.
> <br> 삭제가 간편하다
> - **단점:** 최악의 경우 시간 복잡도가 O(n)이 될 수 있다.
> <br> 데이터가 많아질수록 연결 리스트가 길어져 탐색 속도가 느려진다.
> <br> 연결 리스트를 사용하기 때문에 추가적인 메모리 공간이 필요하다.
>
> **A2:** 개방 주소법을 사용하여 충돌된 데이터를 저장한다. (개방 주소법 / Open Addressing)
> - **장점:** 데이터를 저장하기 위한 배열 외에 추가적인 메모리 공간이 필요하지 않다.
> <br>데이터가 메모리상에 연속적으로 배치되기 때문에, CPU가 데이터를 읽어오는 속도가 체이닝보다 빠르다.
> - **단점:** 데이터를 삭제할 때 단순히 지우기만 하면, 충돌로 인해 뒤에 저장된 데이터를 찾지 못하는 문제가 생길 수 있어 별도의 처리가 필요하다.
> <br> 데이터가 특정 영역에 뭉치는 현상이 발생할 수 있으며, 이 경우 빈자리를 찾기 어려워져 성능이 저하될 수 있다.

### 3-2. 재해싱 (Rehashing)

> **Q:** 재해싱이란 무엇인가?
>
> **A:** 해시 테이블의 크기를 늘려 해시 충돌을 줄이는 방법이다. 이후 데이터들의 위치를 다시 계산하여 재배치한다.

> **Q:** 재해싱은 언제 필요한가?
>
> **A:** 해시 테이블의 크기가 일정 수준 이상으로 늘어나면 해시 충돌이 발생할 확률이 높아지므로, 해시 테이블의 크기를 늘려 해시 충돌을 줄여야 한다.

> **Q:** 재해싱의 시간 복잡도는 어떻게 되는가?
>
> **A:** O(n)이다.
