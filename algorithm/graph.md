# 그래프 알고리즘 (Graph Algorithm)

## 그래프 탐색의 기초

### BFS (너비 우선 탐색)
- 자료구조: 큐 (Queue, FIFO)
- 특징: 시작점에서 가까운 이웃부터 방문.
- 주요 용도: 가중치가 없는 그래프에서의 최단 경로 찾기.

### DFS (깊이 우선 탐색)
- 자료구조: 스택 (Stack, LIFO) 또는 재귀 호출
- 특징: 한 길을 끝까지 파고든 후 막히면 되돌아옴 (백트래킹).
- 주요 용도: 모든 경로 탐색, 그래프의 사이클 확인, 연결 요소 찾기.

## 최단 경로 (Shortest Path)

### 다익스트라 (Dijkstra) 알고리즘
- 자료구조: 우선순위 큐 (Priority Queue)
- 특징: 가중치가 있는 그래프에서 특정 시작점으로부터 모든 정점까지의 최단 거리를 구함.
- 핵심: "현재까지 확인된 가장 짧은 경로"를 가진 정점부터 확장하는 그리디(Greedy) 방식.

## 최소 신장 트리 (MST)

모든 정점을 연결하되, 간선 가중치의 합을 최소로 하는 트리 구조를 찾는 알고리즘.

### 크루스칼 (Kruskal) 알고리즘
- 방식: 간선 중심. 모든 간선을 가중치 순으로 정렬 후 하나씩 선택.
- 핵심 기술: **유니온-파인드 (Union-Find)**를 사용해 사이클 발생 여부 확인.
- 적합한 상황: 간선이 적은 희소 그래프(Sparse Graph).

### 프림 (Prim) 알고리즘
- 방식: 정점 중심. 시작 정점에서 세력을 확장하며 가장 싼 간선 선택.
- 핵심 기술: 우선순위 큐를 사용해 연결 가능한 최저 비용 간선 추출.
- 적합한 상황: 간선이 빽빽한 밀집 그래프(Dense Graph).

## 유니온-파인드 (Union-Find)
- 목적: 서로소 집합(Disjoint Set) 관리 및 사이클 판별.
- 연산:
  - Find: 특정 원소의 대표 원소(팀장)를 찾음. (경로 압축으로 최적화)
  - Union: 두 그룹을 하나로 합침.

## 위상 정렬 (Topological Sort)
- 대상: DAG (방향성이 있고 사이클이 없는 그래프).
- 핵심 개념: 진입 차수 (Indegree).
- 방식: 진입 차수가 0인 노드를 큐에 넣고 연쇄적으로 제거하며 순서 나열.
- 용도: 선수과목 이수 체계, 작업 스케줄링, 사이클 존재 여부 판별.
