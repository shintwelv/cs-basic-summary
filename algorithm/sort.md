# 정렬 (Sort)

데이터를 특정 기준(오름차순, 내림차순)에 따라 나열하는 알고리즘이다.

## 1. 정렬 알고리즘 요약

| 분류 | 알고리즘 | 시간 복잡도 (평균) | 특징 |
| :--- | :--- | :--- | :--- |
| **기초 (O(n²))** | 버블 정렬 | $O(n^2)$ | 구현이 매우 쉬우나 성능이 가장 낮음 |
| | 선택 정렬 | $O(n^2)$ | 데이터 교환(Swap) 횟수가 최소임 |
| | 삽입 정렬 | $O(n^2)$ | 이미 정렬된 데이터에선 $O(n)$으로 매우 빠름 |
| **심화 (O(n log n))** | 병합 정렬 | $O(n \log n)$ | 항상 일정한 성능을 보장하지만 추가 메모리가 필요함 |
| | 퀵 정렬 | $O(n \log n)$ | 평균적으로 가장 빠르나 최악의 경우 $O(n^2)$ 발생 |
| | 힙 정렬 | $O(n \log n)$ | 추가 메모리 없이 최악의 경우에도 안정적인 성능 |

---

## 2. 주요 정렬 알고리즘 상세

> **Q: 버블 정렬(Bubble Sort)의 원리와 장단점은 무엇인가?**
>
> **A:** 인접한 두 원소를 비교하며 큰 값을 뒤로 보내는 과정을 반복한다. 마치 거품이 올라오는 것과 같아 버블 정렬이라 불린다.
> - **장점:** 구현이 매우 직관적이고 간단하다.
> - **단점:** 성능이 매우 좋지 않아 실제 서비스에서는 거의 사용되지 않는다.

> **Q: 선택 정렬(Selection Sort)이 버블 정렬보다 나은 점은 무엇인가?**
>
> **A:** 매 단계에서 가장 작은 값을 '선택'하여 맨 앞과 교환한다.
> - **특징:** 버블 정렬에 비해 실제로 값을 바꾸는 **교환(Swap) 횟수**가 적어, 쓰기 작업이 비싼 시스템에서 유리할 수 있다. 하지만 비교 횟수는 여전히 $O(n^2)$이다.

> **Q: 삽입 정렬(Insertion Sort)은 언제 가장 효율적인가?**
>
> **A:** 카드를 정렬하듯 데이터를 하나씩 적절한 위치에 '삽입'하는 방식이다.
> - **특징:** **거의 정렬된 상태**의 데이터에서는 비교를 거의 하지 않아 $O(n)$의 속도를 보여준다. 성능이 $O(n \log n)$인 알고리즘들보다도 빠를 수 있어, 하이브리드 정렬 알고리즘(예: Timsort)의 기반이 된다.

## 3. 심화 정렬 알고리즘

> **Q: 병합 정렬(Merge Sort)의 핵심 철학은 무엇인가?**
>
> **A:** **분할 정복(Divide and Conquer)** 이다. 문제를 계속 반으로 쪼갠 뒤 다시 합치면서 정렬한다.
> - **장점:** 데이터 분포와 상관없이 항상 $O(n \log n)$을 보장하는 안정적인 알고리즘이다.
> - **단점:** 합치는 과정에서 데이터를 저장할 **추가적인 메모리 공간**이 필요하다.

> **Q: 퀵 정렬(Quick Sort)이 가장 널리 쓰이는 이유는 무엇인가?**
>
> **A:** 기준점(Pivot)을 설정하고 이를 기준으로 데이터를 작게 쪼개는 방식이다.
> - **장점:** 평균적으로 $O(n \log n)$ 중에서도 가장 성능이 좋다. (캐시 효율성 등 하드웨어 친화적)
> - **단점:** 기준점이 계속 최솟값이나 최댓값으로 잡히는 최악의 경우 $O(n^2)$이 될 수 있다.

> **Q: 힙 정렬(Heap Sort)은 어떤 경우에 유리한가?**
>
> **A:** 완전 이진 트리 구조인 **힙(Heap)** 을 이용하는 방식이다.
> - **장점:** 병합 정렬처럼 최악의 경우에도 $O(n \log n)$을 보장하면서, 병합 정렬과 달리 추가 메모리를 사용하지 않는다(In-place).
