# 시간 및 공간 복잡도 (Time & Space Complexity)

## 1. 복잡도 분석 기초

> **Q:** 시간 복잡도와 공간 복잡도의 차이는 무엇인가?

> **A:** **시간 복잡도** 는 알고리즘이 실행되는 데 걸리는 **시간(연산 횟수)** 을 의미하고, **공간 복잡도** 는 실행에 필요한 **메모리 양** 을 의미한다.

> **Q:** 빅오 표기법에서 계수나 상수를 무시하고 $O(n)$으로 표현하는 이유는 무엇인가?

> **A:** 데이터의 양($n$)이 무한히 커질 때, 상수는 전체적인 **증가 추세(그래프의 모양)** 에 큰 영향을 주지 않기 때문이다. 즉, 알고리즘의 "급"을 결정하는 가장 지배적인 항만 남기는 것이다.

## 2. 알고리즘과 효율성

> **Q:** 공간 복잡도 관점에서 효율성이 다른 두 방식을 비교하라.

> **A:** 두 방식의 차이는 추가적인 메모리 사용 여부에 있다.
> - **알고리즘 A (In-place Swap)**: 기존 공간을 활용하므로 추가 메모리가 거의 들지 않아 **공간 복잡도 $O(1)$** 이다.
> - **알고리즘 B (새 배열 생성)**: 입력 크기 $n$만큼의 새로운 공간이 필요하므로 **공간 복잡도 $O(n)$** 이다.
> - **결론**: 두 방식 모두 요소를 한 번씩 건드린다면 시간 복잡도는 $O(n)$으로 같을 수 있지만, 공간 효율성은 A가 훨씬 높다.

## 3. 재귀와 최적화

> **Q:** 재귀 피보나치의 한계와 메모이제이션의 원리는 무엇인가?

> **A:** 
> - **재귀의 문제**: $f(n-1)$과 $f(n-2)$를 호출할 때, 이미 계산했던 하위 문제들을 계속해서 중복 계산하기 때문에 시간이 기하급수적으로 늘어난다 ($O(2^n)$).
> - **메모이제이션(Memoization)의 원리**: 한 번 계산한 값을 배열 등에 저장해 두었다가 다시 꺼내 씀으로써, 중복 계산을 없애고 시간을 선형적으로 줄여준다 ($O(n)$).
